# Public RPC Operator Policy

Status: controlled-testnet policy, 2026-05-13.

This runbook is the source of truth for exposing PostFiat L1 RPC. Validator
processes never bind their plaintext RPC listener to a public or wildcard
address. Public clients terminate TLS and authenticate at a separately managed
edge, which forwards to the validator's loopback listener. Signed write
admission and Orchard batch creation remain explicit, bounded opt-ins.

## Default Service

Provisioned validators run `postfiat-node rpc-serve` from the generated
systemd unit:

```bash
postfiat-node rpc-serve \
  --data-dir /var/lib/postfiat/validator-N \
  --bind-host 127.0.0.1 \
  --port <rpc_port> \
  --max-requests ${POSTFIAT_RPC_MAX_REQUESTS} \
  --timeout-ms ${POSTFIAT_TIMEOUT_MS} \
  --child-timeout-ms ${POSTFIAT_RPC_CHILD_TIMEOUT_MS} \
  --event-log /var/log/postfiat/validator-N/rpc-events.ndjson
```

Controlled-testnet defaults are generated by `scripts/testnet-provision-bundle`:

| Setting | Default | Policy |
| --- | ---: | --- |
| RPC port | `RPC_BASE_PORT + validator_index` (`27650+`) | Open only for approved testnet peers, monitoring, and public read clients. |
| Bind host | `127.0.0.1` | Mandatory. The node rejects public, wildcard, and hostname binds even if the legacy public-bind environment variable is present. |
| `POSTFIAT_RPC_MAX_REQUESTS` | `10000` | Bounded lifecycle cap; systemd restarts the process after clean exhaustion. |
| `POSTFIAT_TIMEOUT_MS` | `30000` | Per-connection socket timeout. |
| `POSTFIAT_RPC_CHILD_TIMEOUT_MS` | `30000` | Child command execution timeout; timeout is reported as `rpc_child_timeout`. |
| Event log | `rpc-events.ndjson` | Required. Ship or retain with validator logs. |
| Transparent write methods | Disabled | Do not pass `--allow-mempool-submit` on public validator RPC by default. |
| Orchard batch creation | Disabled | Do not pass `--allow-orchard-batch-create` on public validator RPC by default. If enabled, only bounded inline JSON batch creation is admitted, direct shielded apply remains disabled, and `--max-orchard-batch-create-concurrent` must remain bounded. |

## Allowed Read Surface

Read-only `rpc-serve` accepts:

- Node and ledger status: `status`, `server_info`, `metrics`, `ledger`.
- Account and transaction reads: `account`, `receipts`, `tx`, `blocks`,
  `batch_archive`.
- Validator/governance reads: `validators`, `manifests`, `verify_state`,
  `verify_blocks`, `verify_bridge`, `verify_mempool`, `verify_shielded`.
- Mempool/shield/bridge reads: `mempool_status`, `shield_scan`,
  `shield_disclose`, `shield_turnstile`, `bridge_status`.
- Fee reads: `fee`, `transfer_fee_quote`.

`manifests` reports `available=false` unless
`governance-genesis-bundle.json` is present in the validator data directory and
its referenced operator manifest files verify. That is intentional: a node must
not invent operator metadata if the canonical bundle is absent.

## Write Opt-In

`mempool_submit_signed_transfer` is the only remote transparent write admitted
by `rpc-serve`, and only when all of the following are true:

- The service is started with `--allow-mempool-submit`.
- `--max-mempool-submit-per-peer` is set to a positive bounded value.
- `--max-mempool-submit-total` is set to a positive bounded value.
- Invalid-signature, duplicate-transaction, peer-limit, and global-limit
  counters are monitored from the `postfiat-rpc-serve-v1` report.
- A valid read after pressure is tested before the endpoint is advertised.

Use a dedicated authenticated write edge. Do not expose the validator process
directly as a fallback.

## Orchard Batch-Creation Opt-In

`shield_batch_orchard` and `shield_batch_orchard_withdraw` are admitted by
`rpc-serve` only when the service is started with
`--allow-orchard-batch-create`.

This mode is deliberately narrower than public shielded apply:

- Requests must carry bounded `action_json`; server-local `action_file` and
  client-selected `batch_file` paths are rejected.
- The server writes action/batch artifacts only under its controlled spool
  directories.
- Each request runs through the existing `--child-timeout-ms` child-process
  boundary.
- Operators must set positive `--max-orchard-batch-create-per-peer` and
  `--max-orchard-batch-create-total` limits.
- Operators must set positive `--max-orchard-batch-create-concurrent`; the
  controlled-testnet default is `1`, so excess simultaneous verifier work fails
  before a child verifier starts.
- `apply_shield_batch` is not admitted by `rpc-serve`.

## Logs And Retention

Operators must retain:

- `/var/log/postfiat/validator-N/rpc-events.ndjson`
- `/var/log/postfiat/validator-N/stdout.log`
- `/var/log/postfiat/validator-N/stderr.log`
- systemd journal entries for `postfiat-validator-N-rpc.service`

Minimum retention for controlled testnet is the greater of seven days or the
current evidence window. If partial-history pruning is enabled, retain RPC logs
through the archive handoff window so `tx`, `ledger`, and `history-status`
answers can be correlated with archive evidence.

## Edge And Firewall Posture

Do not open the validator RPC port publicly. A public load balancer or reverse
proxy must reach it over loopback, a Unix socket, or a private authenticated
overlay and must enforce:

- connection and request-rate limits;
- request body limits at or below `MAX_RPC_REQUEST_BYTES` (`65536`);
- TLS termination for public Internet access;
- access logs with source IP, method, status, and response class;
- no request rewriting that changes JSON request bodies.

Validator-to-validator plaintext transport likewise binds only to explicit
private-overlay IP addresses. Public topology IPs, DNS names, and wildcard
binds fail closed during deployment staging and process startup. Public routing
to that overlay is an operator misconfiguration, not a supported deployment.

## Verification Commands

Before advertising an endpoint:

```bash
scripts/testnet-rpc-read-alias-smoke
scripts/testnet-rpc-serve-tamper
scripts/testnet-rpc-write-edge-load-smoke   # only for write-edge services
```

The readiness gate requires the read-alias smoke and the P0 gate refuses a
local readiness report that omits it.

## Current Gaps

- Production edge authentication, TLS automation, and external rate-limiter
